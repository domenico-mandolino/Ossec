# Ossec
# # Mise en place d'un système HIDS avec OSSEC
# # # Introduction du sujet

Objectifs

● Comprendre le fonctionnement d'un système de détection d'intrusion
sur hôte (HIDS)

● Maîtriser l'installation et la configuration d'OSSEC en mode
client/serveur

● Apprendre à configurer des règles de détection et à analyser les alertes

● Comprendre l'intérêt d'un HIDS dans une stratégie globale de gestion
des risques

# # Job 01
Vous devez vous renseigner sur qu'est-ce qu'OSSEC et faire une
documentation de synthèse

# # Job 02
Depuis une VM debian , installer le serveur OSSEC depuis les sources .
Faite une installation "hybride" ( le serveur et l’agent sur la même machine )
Faire la documentation de l’installation

# # Job 03
Installation d’une VM « cliente » , soit sous debian ou sous windows ( si le PC
est assez puissant ) .
Installer l’agent sur la VM

# # Job 04
Sur la VM cliente faites :
- Des simulations de modifications de fichiers système .
- Des simulations de tentatives d'intrusion .
- Des simulations de comportement suspect .
- Des simulations d'installation de rootkit .


# # Pour aller plus loin
Détection et analyse d'une attaque par force brute avec réponse active
Pour cela vous devez :
- Configurer des règles personnalisées avancées dans OSSEC
- Mettre en place une réponse active
- Analyser les logs et comprendre le processus de détection
- Créer un rapport d'incident

# # Compétences visées
➔ Participer à la mesure et à l’analyse du niveau de sécurité de
l’infrastructure
➔ Participer à l’élaboration et à la mise en œuvre de la politique de sécurité
